# 用户购买流程业务文档

## 概述

本文档描述 XinMall 用户购买商品的完整业务流程，包括从商品浏览到订单完成评价的全链路流程。

---

## 流程概览

```
首页 → 搜索/找产品 → 商品详情 → 确认订单 → 支付 → 待发货 → 待收货 → 评价
```

---

## 详细流程说明

### 1. 首页入口

**页面路径**: `pages/index/index`

**功能描述**: 用户进入应用后的首页，展示推荐商品和快捷入口。

**主要功能**:
- 搜索栏：点击进入搜索入口页
- 找产品入口：点击进入产品分类页
- 分类标签：快速筛选商品类别
- 轮播图：展示活动/推荐商品
- 商品瀑布流：展示推荐商品列表

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 点击搜索栏 | 搜索入口页 | `/pages-sub/search/entry` |
| 点击找产品图标 | 产品分类页 | `/pages-sub/search/category` |
| 点击商品卡片 | 商品详情页 | `/pages-sub/trade/product/detail?id={商品ID}` |
| 点击用户头像 | 用户主页 | `/pages-sub/content/user/index?id={用户ID}` |

---

### 2. 商品搜索

#### 2.1 搜索入口页

**页面路径**: `pages-sub/search/entry`

**功能描述**: 搜索入口页，展示搜索历史和热门搜索。

**主要功能**:
- 搜索输入框
- 搜索历史记录
- 热门搜索关键词

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 输入关键词搜索 | 搜索结果页 | `/pages-sub/search/result?keyword={关键词}` |
| 点击热门关键词 | 搜索结果页 | `/pages-sub/search/result?keyword={关键词}` |

#### 2.2 找产品页（产品分类）

**页面路径**: `pages-sub/search/category`

**功能描述**: 按设备类型和品牌分类查找产品，展示确定型号的商品。

**主要功能**:
- 设备类型网格（手机、电脑、平板、耳机等）
- 品牌字母索引列表
- 快速搜索入口

**业务说明**:
> 通过"找产品"进入的搜索页面，展示的是确定型号的商品列表。这些商品来自商家发布的标准化产品，而非个人闲置发布。

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 点击设备类型 | 设备类型页 | `/pages-sub/search/device-type?id={类型ID}&name={类型名称}` |
| 点击品牌分类 | 设备类型页 | `/pages-sub/search/device-type?id={分类ID}&name={分类名称}` |

#### 2.3 搜索结果页

**页面路径**: `pages-sub/search/result`

**功能描述**: 展示搜索结果商品列表。

**主要功能**:
- 商品列表展示
- 筛选排序功能
- 价格区间筛选

---

### 3. 商品详情页

**页面路径**: `pages-sub/trade/product/detail`

**功能描述**: 展示单个商品的详细信息，用户可在此页面决定是否购买。

**商品类型说明**:

| 类型 | 来源 | 特点 |
|------|------|------|
| 个人闲置 | 个人用户发布 | 一件一拍，独特商品 |
| 商家商品 | 商家店铺发布 | 标准化产品，库存管理 |

**主要功能**:
- 商品图片轮播
- 商品信息（价格、标题、成色、规格）
- 卖家信息卡片
- 商品描述
- 收藏功能
- 私信功能
- 联系卖家
- 立即购买

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 点击立即购买 | 确认订单页 | `/pages-sub/trade/order/confirm?productId={商品ID}` |
| 点击联系卖家 | 聊天页面 | `/pages-sub/chat/index?userId={卖家ID}` |
| 点击收藏 | - | 本地状态更新 |

---

### 4. 确认订单页

**页面路径**: `pages-sub/trade/order/confirm`

**功能描述**: 用户确认订单信息并提交订单。

**主要功能**:
- 收货地址选择/添加
- 商品信息确认
- 配送方式选择
- 订单备注
- 价格明细（商品总额、运费、合计）

**业务规则**:
1. 必须选择收货地址才能提交订单
2. 提交订单后跳转支付页面
3. 订单创建后状态为"待付款"

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 点击选择地址 | 地址列表页 | `/pages-sub/user/address/list?select=1` |
| 提交订单成功 | 支付页面 | `/pages-sub/trade/pay/index?id={订单ID}` |

---

### 5. 支付页面

**页面路径**: `pages-sub/trade/pay/index`

**功能描述**: 用户选择支付方式完成付款。

**主要功能**:
- 支付金额展示
- 订单信息展示
- 支付方式选择（微信支付、支付宝）
- 支付倒计时（30分钟）

**业务规则**:
1. 订单创建后30分钟内需完成支付
2. 支付成功后订单状态变更为"待发货"
3. 超时未支付订单自动取消

**支付流程**:
```
选择支付方式 → 点击支付 → 调起支付SDK → 支付结果 → 跳转订单详情
```

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 支付成功 | 订单详情页 | `/pages-sub/trade/order/detail?id={订单ID}` |
| 支付取消/失败 | - | 留在当前页，提示重试 |

---

### 6. 订单状态流转

#### 6.1 订单列表页

**页面路径**: `pages-sub/trade/order/list`

**功能描述**: 展示用户所有订单，支持按状态筛选。

**订单状态**:

| 状态 | 说明 | 用户操作 |
|------|------|----------|
| pending | 待付款 | 立即付款、取消订单 |
| paid | 待发货 | 联系卖家、申请退款 |
| shipped | 待收货 | 确认收货、查看物流 |
| completed | 待评价 | 去评价、再次购买 |
| evaluated | 已完成 | 再次购买、删除订单 |
| cancelled | 已取消 | 删除订单 |

**跳转逻辑**:

| 操作 | 目标页面 | 路由 |
|------|----------|------|
| 点击订单卡片 | 订单详情页 | `/pages-sub/trade/order/detail?id={订单ID}` |
| 点击立即付款 | 支付页面 | `/pages-sub/trade/pay/index?id={订单ID}` |
| 点击确认收货 | - | 弹窗确认，更新状态 |
| 点击去评价 | 评价页面 | `/pages-sub/trade/evaluate/index?id={订单ID}` |

#### 6.2 订单详情页

**页面路径**: `pages-sub/trade/order/detail`

**功能描述**: 展示订单完整信息和操作入口。

**主要功能**:
- 订单状态跟踪
- 物流信息
- 商品信息
- 价格明细
- 操作按钮

---

### 7. 确认收货

**触发条件**: 订单状态为"待发货"（shipped）

**操作流程**:
1. 用户在订单列表或详情页点击"确认收货"
2. 弹窗确认："确认已收到商品吗？"
3. 确认后订单状态变更为"待评价"（completed）

**业务规则**:
- 卖家发货后用户可确认收货
- 确认收货后资金打款给卖家
- 确认收货后可进行评价

---

### 8. 评价页面

**页面路径**: `pages-sub/trade/evaluate/index`

**功能描述**: 用户对已完成订单进行评价。

**主要功能**:
- 商品评分（描述相符、物流服务、服务态度）
- 评价内容输入
- 图片上传（选填，最多9张）
- 快捷标签选择

**业务规则**:
1. 评价内容至少10个字
2. 评分默认5星，可调整
3. 提交评价后订单状态变更为"已评价"（evaluated）
4. 每个订单只能评价一次

**评价内容**:

| 评价项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 描述相符 | 星级评分 | 是 | 1-5星 |
| 物流服务 | 星级评分 | 是 | 1-5星 |
| 服务态度 | 星级评分 | 是 | 1-5星 |
| 评价内容 | 文本 | 是 | 至少10字，最多500字 |
| 图片 | 图片数组 | 否 | 最多9张 |
| 快捷标签 | 标签数组 | 否 | 多选 |

## 商品类型对比

### 个人闲置 vs 商家商品

| 特性 | 个人闲置 | 商家商品 |
|------|----------|----------|
| 发布者 | 个人用户 | 认证商家 |
| 商品数量 | 一件一拍 | 有库存 |
| 商品来源 | "找产品"搜索 | 首页推荐/搜索 |
| 价格 | 个人定价 | 商家定价 |
| 售后 | 有限 | 完善 |
| 发票 | 可能有 | 有 |
| 质保 | 视情况 | 有 |

### 入口区分

| 入口 | 展示内容 | 页面路径 |
|------|----------|----------|
| 首页搜索 | 混合商品 | `/pages-sub/search/result` |
| 首页找产品 | 确定型号商品 | `/pages-sub/search/category` → `/pages-sub/search/device-type` |
| 首页推荐 | 混合商品 | 直接进入详情 |

## 关键业务规则

### 订单相关

1. **订单有效期**: 待付款订单30分钟后自动取消
2. **支付超时**: 支付页面倒计时30分钟
3. **自动确认**: 发货后7天自动确认收货
4. **评价期限**: 确认收货后15天内可评价
5. **退款期限**: 发货前可申请退款

### 价格相关

1. **商品价格**: 卖家发布时设置
2. **运费**: 默认包邮，特殊商品除外
3. **优惠**: 暂不支持优惠券

### 地址相关

1. **默认地址**: 用户可设置默认收货地址
2. **地址数量**: 最多保存20个地址
3. **地址信息**: 姓名、手机号、详细地址
