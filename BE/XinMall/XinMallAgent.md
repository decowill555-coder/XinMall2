# XinMallAgent 自动化测试系统

## 系统架构

### 1. 核心理念
XinMallAgent 是一个自动化测试和修复系统，能够：
- 自主探索应用程序界面
- 生成测试用例和预期行为
- 执行测试并验证结果
- 自动诊断和修复问题
- 循环运行直到所有问题解决

### 2. 代理协作模式

```
┌─────────────────────────────────────────────────────────────┐
│                    XinMallAgent 主控                        │
│  (协调各代理，管理测试循环，追踪问题和修复)                    │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ 测试规划代理   │    │ 测试执行代理   │    │ 问题诊断代理   │
│ TestPlanner   │    │ TestExecutor  │    │ Diagnostician │
└───────────────┘    └───────────────┘    └───────────────┘
        │                     │                     │
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│ - 页面分析     │    │ - 操作执行     │    │ - 差异分析     │
│ - 用例生成     │    │ - 结果验证     │    │ - 根因定位     │
│ - 预期定义     │    │ - 状态记录     │    │ - 修复建议     │
└───────────────┘    └───────────────┘    └───────────────┘
```

### 3. 测试流程

```
1. 发现阶段
   ├── 获取当前页面快照
   ├── 分析页面元素和功能
   └── 识别可测试的交互点

2. 规划阶段
   ├── 为每个交互点生成测试用例
   ├── 定义预期行为
   └── 确定验证标准

3. 执行阶段
   ├── 执行测试操作
   ├── 记录实际行为
   └── 比对预期与实际

4. 诊断阶段
   ├── 发现差异时分析原因
   ├── 定位问题代码
   └── 制定修复方案

5. 修复阶段
   ├── 应用代码修复
   ├── 重新测试验证
   └── 记录修复结果
```

### 4. 测试用例模板

每个测试用例包含：
```json
{
  "testId": "TC-XXX",
  "description": "测试描述",
  "preconditions": ["前置条件"],
  "steps": [
    {"action": "click/input/navigate", "target": "元素标识", "value": "输入值"}
  ],
  "expectedResults": [
    {"type": "url/text/state/network", "value": "预期值"}
  ],
  "priority": "P0/P1/P2",
  "category": "功能分类"
}
```

### 5. 问题追踪模板

```json
{
  "issueId": "ISSUE-XXX",
  "description": "问题描述",
  "severity": "critical/major/minor",
  "location": "文件路径:行号",
  "expectedBehavior": "预期行为",
  "actualBehavior": "实际行为",
  "rootCause": "根本原因分析",
  "fixApplied": "应用的修复",
  "status": "open/fixed/verified"
}
```

## 运行模式

### 连续测试模式
- 系统将持续运行，自动发现和修复问题
- 每轮测试覆盖不同模块
- 优先处理高优先级问题

### 覆盖范围
1. **用户认证模块**: 注册、登录、密码找回
2. **商品模块**: 列表、详情、搜索、筛选
3. **购物车模块**: 添加、修改、删除、结算
4. **订单模块**: 创建、支付、状态流转
5. **用户中心模块**: 个人信息、地址、收藏
6. **社区模块**: 帖子、评论、点赞
7. **消息模块**: 通知、未读状态

## 指令格式

### 启动测试规划代理
```
启动 TestPlanner 代理，分析当前页面并生成测试用例
```

### 启动测试执行代理
```
启动 TestExecutor 代理，执行指定测试用例
```

### 启动诊断代理
```
启动 Diagnostician 代理，分析问题并提供修复建议
```
